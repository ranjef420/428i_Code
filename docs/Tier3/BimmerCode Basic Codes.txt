Acoustics/Confirmaton
-----------------------------------
1) Acoustic Pedestrian Protection – Checkbox in iDrive
Purpose: Expose the OEM checkbox to enable/disable acoustic pedestrian protection in iDrive.
Coding Type: FDL
ECU/Module: HU_NBT_EVO (Headunit)
Prerequisites: Vehicle hardware supports acoustic pedestrian protection.
iDrive Impact: Adds/retains a toggle in Settings; does not itself force the feature on/off unless toggled in iDrive.
Values: Active | Not active
Verify: iDrive → Settings → Sound/Driving Assistance (path may vary) → Checkbox visible and functional
Rollback/Notes: Revert FDL to default. Check local regulations for use.

2) Acoustical Confirmation (Lock/Unlock Chirp)
Purpose: Enable/disable acoustic confirmation when locking/unlocking.
Coding Type: FDL
ECU/Module: FZD (Roof Function Center) or Alarm-related ECU (confirm exact name as shown in your tool)
Prerequisites: Alarm/confirmation hardware fitted.
iDrive Impact: Controls behavior; may not expose a checkbox unless separately enabled.
Values: Active | Not active
Verify: Lock/unlock with key fob; listen for chirp per setting
Rollback/Notes: If no chirp after activation, confirm alarm presence and country restrictions.

3) Checkbox – Acoustical Lock Confirmation (Expose UI Toggle)
Purpose: Expose an iDrive checkbox to allow the driver to enable/disable lock confirmation.
Coding Type: FDL
ECU/Module: HU_CIC or HU_NBT or HU_NBT_EVO (Headunit)
Prerequisites: OEM alarm system; FZD/Alarm ECU acoustical confirmation should also be Active
iDrive Impact: Adds checkbox in Doors/Key settings; user controls chirp thereafter
Values: Active | Not active
Verify: iDrive → Settings → Doors/Key → “Acoustic confirmation” appears and is selectable
Rollback/Notes: If checkbox is gray/inactive, set acoustical confirmation Active in FZD/Alarm ECU.

4) Confirmation Sound – Duration
Purpose: Set duration of lock/unlock confirmation sound.
Coding Type: FDL
ECU/Module: FZD (Roof Function Center) or Alarm-related ECU
Prerequisites: Acoustical confirmation enabled and functional
iDrive Impact: No new menu item; behavior changes immediately
Values: Short | Normal | Long | Very long
Verify: Lock/unlock and compare chirp length
Rollback/Notes: Restore to Normal if unsure.

5) Confirmation Sound – Frequency
Purpose: Set frequency (pitch) of the confirmation sound.
Coding Type: FDL
ECU/Module: FZD (Roof Function Center) or Alarm-related ECU
Prerequisites: Acoustical confirmation enabled and functional
iDrive Impact: No new menu item; behavior changes immediately
Values: Low | Normal | High
Verify: Lock/unlock and compare pitch
Rollback/Notes: Some markets limit audible confirmations.

6) Confirmation Sound – Volume
Purpose: Set volume of the confirmation sound.
Coding Type: FDL
ECU/Module: FZD (Roof Function Center) or Alarm-related ECU
Prerequisites: Acoustical confirmation enabled and functional
iDrive Impact: No new menu item; behavior changes immediately
Values: Low | Normal | High | Very high
Verify: Lock/unlock and compare loudness
Rollback/Notes: Use prudent settings in residential areas.

7) Active Cruise Control – Default Distance
Purpose: Set default following distance level when ACC is activated.
Coding Type: FDL
ECU/Module: ICMQL (Integrated Chassis Management)
Prerequisites: Vehicle equipped with Active Cruise Control hardware
iDrive Impact: Default distance on activation; driver can still adjust during use
Values: Level 1 | Level 2 | Level 3 | Level 4  (Level numbering per ECU mapping)
Verify: Engage ACC; observe default gap indicator
Rollback/Notes: If nonfunctional, verify ACC option code and radar presence.

8) Active Sound Design – Deactivation
Purpose: Disable Active Sound Design engine sound augmentation.
Coding Type: FDL
ECU/Module: ASD_01 (Active Sound Design)
Prerequisites: ASD fitted
iDrive Impact: No menu item; behavior changes immediately
Values: Deactivated (or Not active, per parameter naming)
Verify: Compare interior engine sound before/after under load
Rollback/Notes: Make and retain a full NCD/CAF backup before change; some firmware requires restoring the exact backup to re-enable.
